Clean Pass Dashboard - Changes Made
==================================

Backend Changes:
----------------
1. washstation/views.py:
   - Updated WorkerViewSet to list workers with extra stats (washesThisMonth, revenueThisMonth, commissionEarned, shiftsThisWeek, activeShift)
   - Updated CustomerViewSet to include customers with their vehicles
   - Updated VehicleViewSet to list vehicles with extra fields (activeWashes, unpaidWashes, isRewardReady) and support partial updates (mark paid, adjust washes)
   - Updated AppointmentViewSet to list appointments with date filtering
   - Added WorkerShiftView to handle clock in/clock out actions
   - Updated LiveWashQueueView to return detailed wash cards with vehicle, package, worker info and support PATCH for updating wash status/worker
   - Added WashRequestsView to handle approving/rejecting wash requests and creating new washes, with error handling for missing packages
   - Fixed WashRequestsView: Removed 'customer' argument from Wash.objects.create() as Wash model doesn't have a customer field
   - Updated VehicleViewSet to handle cases where vehicleType or customer are null by providing default fallback objects
   - Added PublicCustomerView: Public endpoint to fetch customer's loyalty card info, uses shopId as path param, returns exists, vehicle, customer, activeWashes, washGoal, isRewardReady, and vehicles list
   - Added PublicTrackView: Public endpoint to track wash status, uses shopId as path param, returns activeWash, queuePosition, activeStamps, washGoal, isRewardReady, recentWashes
   - Updated PublicCustomerView to include more data: totalWashes, totalRedemptions, unpaidCount, history, and shop info for MyCard view

2. config/urls.py:
   - Added WorkerShiftView at /api/workers/shift/
   - Added WashRequestsView at /api/requests/ and /api/requests/<uuid:pk>/
   - Updated LiveWashQueueView to support uuid pk in URL
   - Added PublicCustomerView at /api/public/customer/<uuid:shop_id>/
   - Added PublicTrackView at /api/public/track/<uuid:shop_id>/

3. washstation/serializers.py:
   - Updated WashRequestSerializer to return fields in camelCase (plateNo instead of plate_no, createdAt instead of created_at) to match frontend expectations
   - Updated WashRequestSerializer to include package details (WashPackageSerializer)
   - Updated VehicleTypeSerializer to return washGoal instead of wash_goal
   - Updated ShiftSerializer to return clockIn and clockOut instead of clock_in and clock_out
   - Updated CustomerSerializer to return createdAt instead of created_at
   - Updated VehicleSerializer to return plateNo, vehicleType, createdAt instead of plate_no, vehicle_type, created_at
   - Updated WashSerializer to return paymentMethod, createdAt, washStartAt, washDoneAt instead of payment_method, created_at, wash_start_at, wash_done_at
   - Updated AppointmentSerializer to return timeSlot instead of time_slot, added vehicle, package, and createdAt
